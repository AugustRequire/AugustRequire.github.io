关于HALO CE的服务器搭建

这个搭建的方法是我在油管上偶然找到的，虽然它的过程有些繁琐，能够取代它的更简便方法也不是没有，我还是准备把它挂在这里作为一个笔记。毕竟，这个方法是完成着最容易的一个。

搭建步骤

运行 haloceded.exe

这个文件是CE附带的开服工具，想要维持服务器的运行就要保证这个文件处于运行中。
最初运行 haloceded.exe 后，你的服务器会以默认状态启动，你将会在 haloceded.exe 中看到这样的信息。
> WARNING: sv_name unspecified in init.txt, defaulting to Halo
sv_maxplayers: 16
sv_public_value: 1
halo(
这代表你的服务器正在以默认的 halo 设定下运行着，为了完善你的服务器内容，需要对它进行一些进一步的设定。

服务器的设置指令

你的服务器名称

服务器名称即在服务器列表中所显示的名称，不要使用中文，那样会导致名称显示出错。
在运行中的 haloceded.exe 中输入以下指令

  • 若成功，则会得到以下反馈

    sv_name: [name]
    这个指令用来改变你的服务器名称，将 [name]中的内容更换为你希望的名称，例如：
    sv_name SparrowCookies

    服务器公开显示

    这个指令决定了你的服务器是否公开显示，在运行中的 haloceded.exe 中输入以下指令

  • 若成功，则会得到以下反馈

    true

    服务器最大人数

    这个指令决定了你的多人游戏可以容纳几个玩家，反正这个dead game也没有什么活人了随便设定一个就好了（暴论）
    在运行中的 haloceded.exe 中输入以下指令

  • 若成功，则会得到以下反馈

    sv_maxplayers:[palyer]

    服务器密码

    为你的多人游戏设定一个密码，能有一起游玩HALO CE的同伴还真是令人羡慕啊（
    在运行中的 haloceded.exe 中输入以下指令

  • 如果不希望设置密码，则不必填写 [password]

  • 若成功，则会得到以下反馈

    sv_password:[password](存在密码）
    sv_password: (密码为空）

    mapcycle

    一轮时间

    这个指令规定了你的游戏一轮需要多长时间，在运行中的 haloceded.exe 中输入以下指令

  • 若成功，则会得到以下反馈

    [time]

    添加地图与模式

    使用这个指令添加一个或多个游戏地图及所对应的游戏模式

例如，添加一个timberland地图的CTF游戏，其指令为

  • 若成功，则会得到以下反馈

    》timberland CTF

    开始mapcycle

    使用这个指令启动你的mapcycle

  • 若成功，则会得到以下反馈

    Dedicated server is running on map [map] (0 / [palyer] players)

    Use the sv_end_game command to stop the game.

    关闭服务器

    在不需要运行服务器时，输入以下指令停止你的服务器进程

  • 若成功，则会得到以下反馈

    Server is stopping the game…

    Game Complete. Dedicated server is now idle.

    此时，你的服务器将关闭，游戏也会结束。
